发表于 2019-12-16 更新于 2019-12-24 分类于 hexo
本文字数： 5k 阅读时长 ≈ 5 分钟

网站从搭建至今一直白嫖着Github/Coding Page ，但是近期不管GitHub Page还是Coding Page同时抽风，墙内无法使用，于是有了迁移到自己服务器上的想法，也算让吃灰云发挥一下热量，简单记录下迁移过程。

1、服务器环境准备

先说下整体运作流程：之前 hexo 的运作是通过 hexo d 命令将 public 文件夹下的资源文件 push 到 GitHub 或 Coding 上，然后借用网站的 Page 功能显示成网页。现在要部署到腾讯云的话，则是要在服务器上建一个 repository，然后通过 hexo d 将资源文件推到 repository，再在服务器上利用 git hook 将仓库里的文件同步到网站根目录，而 nginx 的作用是反向代理。
